Node.js - من الصفر إلى الاحتراف

مقدمة في Node.js

Node.js هي بيئة تشغيل تسمح بتنفيذ JavaScript على الخادم (server-side). تُستخدم لبناء تطبيقات الشبكة والخوادم بشكل فعال وسريع.

إنشاء خادم بسيط في Node.js

في Node.js، يمكن إنشاء خادم بسيط باستخدام وحدة http:

const http = require('http');

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('مرحباً بك في Node.js!');
});

server.listen(3000, '127.0.0.1', () => {
  console.log('الخادم يعمل على http://127.0.0.1:3000/');
});

نظام الحزم npm

npm هو مدير الحزم الافتراضي لـ Node.js:

// تثبيت حزمة
npm install express

// إنشاء ملف package.json
npm init -y

Express.js

Express.js هو إطار عمل شائع لتطوير تطبيقات الويب باستخدام Node.js:

const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('مرحباً بك في تطبيق Express!');
});

app.listen(3000, () => {
  console.log('التطبيق يعمل على منفذ 3000');
});
ملاحظة: Node.js يستخدم نموذج I/O غير متزامن (asynchronous) مما يجعله سريعًا.

نظام الملفات (File System)

تُستخدم وحدة fs للتعامل مع الملفات في Node.js:

const fs = require('fs');

// قراءة ملف
fs.readFile('test.txt', 'utf8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// كتابة ملف
fs.writeFile('test.txt', 'مرحباً بالعالم!', (err) => {
  if (err) throw err;
  console.log('تمت كتابة الملف!');
});
تحذير: تأكد من معالجة الأخطاء (error handling) عند التعامل مع العمليات غير المتزامنة.

الاستنتاج

Node.js أداة قوية لبناء تطبيقات الخادم باستخدام JavaScript. مع الاستمرار في التعلم والممارسة، يمكنك إتقان هذه التقنية وبناء تطبيقات مذهلة.
